Chore Helper — MVP Technical Spec (Replit Web App)

1) Goal
Build a simple “real life helper” app for a 6- and 8-year-old that:
	•	makes chores feel like a game
	•	uses checklists + points
	•	unlocks badges
	•	includes a simple store where points can be spent

MVP should be fun in under 60 seconds after opening.

⸻

2) MVP Scope (What we will build first)

Core Features (MVP)
	1.	Checklist (Chores)
	•	Show a list of chores (predefined)
	•	Each chore has a point value
	•	Tap a chore to mark complete / incomplete
	2.	Points
	•	Completing chores adds points
	•	Unchecking removes points (simple anti-cheat for now)
	3.	Badges
	•	Earn badges at point thresholds (ex: 50, 150, 300)
	4.	Store
	•	List of rewards (“Arcade”, “Food Treat”, “Build a House”, etc.)
	•	Each reward costs points
	•	User can “buy” rewards, which subtract points and log purchases
	5.	Persistence
	•	Save points, completed chores, badges earned, and purchases in localStorage
	•	No login, no backend

⸻

3) Non-Goals (Not in MVP)
	•	Photo verification (camera)
	•	Multiple users / profiles
	•	Scheduling engine / calendar integrations
	•	Homework per-page scoring logic
	•	Parent/admin panel
	•	Real-world payment or approvals (we can add “parent approve” later)

6) Points + Badge Rules (MVP Logic)

Points
	•	When a chore becomes completed: points += chore.points
	•	When chore is unchecked: points -= chore.points (floor at 0)

Badge thresholds (example)
	•	50 points → “Starter Badge”
	•	150 points → “Helper Level 2”
	•	300 points → “Chore Master”

Badge evaluation
	•	After any point change, loop through badges:
	•	if points >= threshold and not earned → mark earned + show toast

⸻

7) Technical Implementation (Replit)

Stack
	•	Single-page static web app
	•	index.html + style.css + script.js
	•	No frameworks (keep it kid-readable)

Folder structure

/
  index.html
  style.css
  script.js

Core functions (script.js)
	•	loadState() → read localStorage or create default state
	•	saveState(state) → write localStorage
	•	render() → redraw UI sections based on state
	•	toggleChore(choreId) → update completed + points + badges + save + render
	•	buyItem(itemId) → validate points, subtract, log purchase, save + render
	•	resetDaily() → set all chores to incomplete (optional MVP)

Persistence
	•	localStorage.setItem("choreHelperState", JSON.stringify(state))

9) Acceptance Criteria (Definition of Done)

MVP is “done” when:
	•	App runs in Replit and loads instantly
	•	Kid can:
	•	check a chore
	•	see points go up
	•	unlock at least one badge
	•	buy at least one store item
	•	refresh page and everything is still saved
