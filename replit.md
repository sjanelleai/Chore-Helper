# Chore Helper (HomeQuest)

## Overview
Chore Helper (HomeQuest) is a multi-family gamified chore-tracking application designed for children with robust parent control features. Its primary purpose is to transform mundane household tasks into an engaging experience for kids, while providing parents with comprehensive management tools. The project aims to capture a niche market by offering a unique blend of gamification and essential family management. Key capabilities include customizable chore/reward catalogs, point-based reward redemption, and a secure dual-access model tailored for both parents and children. The long-term ambition is to become the leading solution for family chore management, fostering responsibility and positive habits in children through an enjoyable platform.

## User Preferences
- Kid-friendly, gamified UI with fun fonts and bright colors
- Confetti animation on chore completion
- Mobile-first responsive design

## System Architecture
The application is built on a modern full-stack architecture. The frontend utilizes React with Vite, TanStack Query for data fetching, Wouter for routing, Tailwind CSS for styling, shadcn/ui for UI components, and Framer Motion for animations. The backend is an Express.js server written in TypeScript, primarily serving the frontend and handling specific email functionalities. Supabase, a PostgreSQL-based platform, serves as the database, enforcing multi-family data isolation through Row Level Security (RLS) policies.

Authentication employs a dual model: Supabase Auth handles parent accounts (email/password), while children use a token-based system with a Family Number and PIN. Data access for parents is via the Supabase JS client with RLS, whereas kid-mode uses `kid_*` security definer RPCs to bypass RLS with token validation. SendGrid is integrated for server-side daily parent summary emails. The entire application, frontend and backend, is served together via Vite middleware on port 5000.

Key architectural decisions include:
- **Database-backed Catalog**: Chore and reward definitions are stored per-family in `chore_catalog` and `reward_catalog` tables. These are auto-seeded with default values upon family creation and are fully customizable by parents. Catalog versioning ensures smooth updates of default items.
- **Derived Points System**: Children's points are not stored directly but are dynamically calculated from the `points_ledger` (a detailed audit trail of all point changes). Canonical daily points and family summaries are generated by timezone-safe RPCs (`points_earned_for_day` and `family_daily_summary`).
- **Chore Verification System**: Three approval modes (never/always/smart) control when kids need parent approval before earning points. Smart mode uses a configurable threshold (default 30 points). Per-chore overrides (inherit/always/never) allow fine-grained control. Pending chores are visible in the Parent Panel inbox for approve/reject actions.
- **RPC-centric Mutations**: All data modifications and business logic execution are handled through Supabase Remote Procedure Calls (RPCs) to ensure security, data integrity, and consistent application of business rules.
- **Kid Access System**: A dedicated kid login flow with Family Number and PIN allows children to access their personalized dashboards securely, with sessions managed via `kid_sessions` and `kid_*` RPCs.
- **Parent Portal PIN Gate**: An optional UI-level PIN protects the parent panel on shared devices.
- **Responsive Navigation**: A persistent navigation bar adjusts between a fixed bottom bar on mobile and a collapsible sidebar on desktop, ensuring a consistent user experience.
